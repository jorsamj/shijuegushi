## 1. 产品背景

互动视觉小说适合承载“人生分支”体验：用户不只是阅读故事，而是在关键节点做出选择，并承担由选择、关系、证据和推理共同形成的结果。

《第二人生》的第一版以《雨夜来电》作为开放故事，验证一个轻量但可扩展的互动体验平台：用户从启动页进入人生档案首页，选择一个人生故事，进入剧情页后通过文本推进、选择节点、线索收集、人物关系变化、真相拼图、最终推理和结局报告完成一次完整体验。

第一版不追求后端、账号和商业系统，重点验证：内容组织是否清晰、悬疑人生故事是否有代入感、玩法反馈是否能让玩家感到“真相逐步拼出”。

## 2. 产品定位

第二人生是一款互动视觉小说体验平台。用户可以进入不同人生故事，体验他人的秘密、选择、关系、代价和结局。

第二人生不是单一单一悬疑内容库。第一版开放《雨夜来电》，故事类型为“悬疑人生 / 都市悬疑 / 伪灵异旧案推理”。未来可扩展到悬疑人生、都市人生、爱情人生、职场人生、重启人生、逆转人生、平行人生等多种内容方向。

| 关键词 | 解释 |
| --- | --- |
| 人生档案 | 用户可选择进入的故事入口，不再强调旧的单一内容馆概念 |
| 人生故事 | 平台中的单个可体验内容，可包含章节、选择、关系和结局 |
| 互动视觉小说 | 以文本、场景、角色、选择和反馈为核心体验 |
| 人物关系值 | 选择会影响角色之间的信任、协助、警觉和直面程度 |
| 结局路径报告 | 结局页解释玩家为什么走到该结局 |

## 3. 目标用户

| 用户类型 | 用户特征 | 核心场景 | 关键诉求 |
| --- | --- | --- | --- |
| 都市悬疑读者 | 喜欢旧案、身份疑云、反转和心理压迫 | 晚间体验短篇悬疑人生 | 氛围强、线索公平、结局有因果 |
| 视觉小说玩家 | 熟悉剧情推进、选择分支、回看历史 | 快速进入故事并保存进度 | 阅读顺畅、反馈明确、可存档读档 |
| 互动小说创作者 | 关注内容结构和后续扩写 | 按统一结构接入新故事 | 节点、线索、关系、结局规则稳定 |
| Demo 评审者 | 关注产品形态和开发可行性 | 检查 PRD 是否能指导开发 | 页面结构、数据结构、验收标准可测 |

## 4. 核心卖点

1. 人生档案式入口：以“人生故事”组织内容，未来可接入多类型人生体验。
2. 选择有反馈：选择影响线索、变量、人物关系值、结局路径和结局报告。
3. 真相逐步拼图：核心线索以 10 块真相拼图表达推进感。
4. 阶段成果明确：章节结算、里程碑和成就让玩家感到进展。
5. 结局可复盘：结局页展示线索统计、推理成绩、人物关系、关键选择和路径链。

## 8. 页面功能说明

### 8.1 启动页

展示产品名“第二人生”、产品介绍“欢迎来体验别人的人生。”、补充文案“每一次选择，都会走向另一种人生。”和“开始体验”按钮。

### 8.2 人生档案首页

展示当前可体验的人生故事卡片：

| 故事 | 状态 | 说明 |
| --- | --- | --- |
| 雨夜来电 | 已开放 | 一个暴雨夜，一通来自死者的电话，把你带回三年前的旧案 |
| 旧楼档案 | 未开放 | 每一栋旧楼，都住着不愿被提起的人生 |
| 消失的人 | 未开放 | 有人从生活里消失，也有人从记忆里回来 |

### 8.3 人生档案详情页

展示《雨夜来电》故事组：故事 01《雨夜来电》可开始或继续体验；故事 02《无人接听》、故事 03《第三个声音》显示未完待续。

### 8.4 游戏剧情页

包含场景视觉、角色名牌、剧情文本、继续按钮、选项按钮、工具栏、真相进度和反馈弹窗。工具栏包括：线索、人物、存档、读档、历史、返回人生档案。

## 9. 线索库系统

### 9.1 线索获得

当玩家获得新线索时，系统同时展示 toast 和“发现线索”弹窗。弹窗包含线索名称、分类、描述、是否关键线索、当前线索进度和“收入线索库”按钮。已获得线索不重复弹出。

### 9.2 线索库入口

工具栏“线索”入口显示 `线索 x/15`。存在未查看线索时显示红点或微光提示。打开线索库后清空未读状态。

### 9.3 分类与展示

线索库支持：全部、关键线索、通话、人物、物件、照片、旧案、地点。已获得线索以卡片展示，未获得线索显示“？？？”。关键线索带特殊标记。

### 9.4 真相拼图

真相拼图只统计 10 条核心线索：`clue_dead_call`、`clue_warning_sentence`、`clue_door_woman`、`clue_sister_mark`、`clue_old_phone`、`clue_last_photo`、`clue_photo_background`、`clue_gray_loan`、`clue_zhou_left`、`clue_timed_voice`。获得 5/10、8/10、10/10 时给出轻提示。

### 9.5 线索与结局判断关系

A 结局至少要求关键证据链完整：最后合照、照片背景、灰色借贷、周屿离开、旧语音触发记录，并要求备份证据、选择重启旧案、最终推理达到 4/5。

## 10. 人物关系值系统

人物关系值不是传统“好感度”，而是玩家在故事中形成的关系状态、协助程度、警觉程度和自我面对程度。

| id | 名称 | 含义 | 影响 |
| --- | --- | --- | --- |
| trust_zhuwan | 许知晚：信任 | 林舟对许知晚的信任与许知晚对林舟的配合程度 | 影响许知晚是否更早透露旧手机、姐姐细节、调查动机 |
| support_chenyan | 陈妍：协助 | 陈妍对林舟的支持和介入程度 | 影响是否提示备份、帮助查资料、结局报告中是否形成证据协助 |
| suspicion_zhou | 周屿：警觉 | 周屿对林舟行动的警觉程度 | 影响周屿施压频率、威胁感、是否提前打电话干扰 |
| courage_linzou | 林舟：直面 | 林舟面对旧案和愧疚的程度 | 影响是否更容易选择重启旧案，低值更容易走向 D 逃避 |

### 10.1 初始值与范围

初始值均为 0，范围为 0-100。第一版选择节点按 5 / 10 / 15 / 20 分增减。

### 10.2 示例规则

| 选择 | 关系影响 |
| --- | --- |
| 先不开门但要求证明 | trust_zhuwan +5，courage_linzou +5 |
| 直接让许知晚进屋 | trust_zhuwan +15，suspicion_zhou +5 |
| 给陈妍发消息 | support_chenyan +10 |
| 备份照片 | support_chenyan +5，courage_linzou +10 |
| 接周屿电话 | suspicion_zhou +10 |
| 质问周屿 | courage_linzou +10，suspicion_zhou +10 |
| 删除证据 | courage_linzou -20 |
| 选择重启旧案 | courage_linzou +20 |

### 10.3 人物关系面板

游戏过程中可打开“人物”面板。面板展示许知晚：信任、陈妍：协助、周屿：警觉、林舟：直面。每项展示数值条、当前等级文案和最近一次变化原因。

等级文案：

| 数值 | 许知晚 | 陈妍 | 周屿 | 林舟 |
| --- | --- | --- | --- | --- |
| 0-20 | 疏离 | 微弱 | 未察觉 | 逃避 |
| 21-50 | 试探 | 有限协助 | 警觉 | 动摇 |
| 51-80 | 信任 | 积极协助 | 高度警觉 | 面对 |
| 81-100 | 托付 | 全力协助 | 失控施压 | 直面真相 |

## 11. 章节结算系统

当节点从上一章最后节点进入下一章第一节点时，弹出章节完成反馈。反馈包含本章获得线索、本章关键选择和累计真相进度。如果本章没有线索，展示“本章暂无新线索”。

## 12. 里程碑系统

| 里程碑 | 触发条件 | 反馈文案 |
| --- | --- | --- |
| 命运节点开启 | 获得 clue_dead_call | 你接起了来自三年前的电话。这个夜晚已经无法回到原来的轨道。 |
| 身份疑云推进 | 获得 clue_sister_mark 或 verified_zhuwan_identity = true | 门外的人不再只是陌生人。她知道许知夏留下的秘密。 |
| 旧案裂缝出现 | 获得 clue_gray_loan | 许知夏不是意外坠入黑暗，她曾经试图自救。 |
| 关键物证出现 | 获得 clue_last_photo | 三年前的一张合照，把所有人的谎言重新摆上桌面。 |
| 谎言被拍下 | 获得 clue_photo_background | 照片不会说话，但它记住了周屿不该出现的位置。 |
| 灵异被现实撕开 | 获得 clue_timed_voice 或 understood_dead_call = true | 这不是鬼魂来电，而是一段迟到三年的提醒。 |

## 13. 成就系统

第一版轻量成就包括：雨夜接线人、谨慎的人、第一次相信、证据意识、旧案裂缝、照片会说话、现实解释者、真相重启、无人接听。成就保存在 localStorage 中，解锁时展示 toast，结局页展示本次解锁成就。

## 14. 结局路径报告

结局页不只展示结局文本，还展示本次人生报告：结局名称、结局类型、真相进度、推理成绩、关键线索数量、辅助线索数量、缺失线索数量、关键选择、人物关系值、结局评价和结局路径链。

### 14.1 结局路径链

| 结局 | 路径说明 |
| --- | --- |
| A 真相重启 | 找到最后合照、发现照片背景里的周屿、查到灰色借贷、理解死者来电机制、备份证据、选择重启旧案、最终推理达到 4/5 |
| B 证据失控 | 选择把原始照片交给许知晚，但身份核验、证据备份或来电机制存在缺口，证据链流转失控 |
| C 删除证据 | 选择删除证据且没有建立备份，关键证据消失 |
| D 无人接听 | 未满足重启旧案关键条件，或真相进度、推理成绩、关键选择不足，真相停留在无人接听的电话里 |

## 16. 结局判断规则

结局判断仍保持自动判定，不向玩家直接展示 A/B/C/D 选择。优先级：C 删除证据 > A 真相重启 > B 证据失控 > D 无人接听。

A 结局要求关键线索齐全、已备份照片、选择重启旧案、最终推理 `deductionScore >= 4`。人物关系值用于报告和体验反馈，第一版不直接替代证据链。

## 17. UI 风格

整体风格为暗色玻璃、暖金微光、平行人生、旧照片、雨夜悬疑。用户应感到进入了一个互动视觉小说游戏，而不是普通阅读网页。

## 18. 验收标准

1. README、页面和 PRD 均统一为“第二人生”品牌。
2. 用户可从启动页进入人生档案首页。
3. 用户可进入《雨夜来电》并完成剧情推进。
4. 用户可在游戏过程中打开线索库。
5. 用户可在游戏过程中打开人物关系面板。
6. 人物关系值会随关键选择变化，并写入 localStorage。
7. 旧存档缺少 relationships 时可自动补默认值。
8. 章节切换有阶段成果反馈。
9. 里程碑和成就只触发一次。
10. 结局页展示本次人生报告。
11. 结局页展示“你如何走到这个结局”的路径链。
12. A/B/C/D 四结局均有路径说明。
13. 最终推理仍按 isCorrect 计分，A 结局需要 deductionScore >= 4。
14. 不引入框架，不接后端，不改变《雨夜来电》主线真相。

## 19. 后续迭代方向

1. 增加更多人生故事类型。
2. 为人物关系值增加更多剧情差异反馈。
3. 增加章节内小型交互玩法。
4. 增加音效和真实视觉素材。
5. 增加可视化剧情编辑器。
6. 增加云端存档和跨设备同步。
