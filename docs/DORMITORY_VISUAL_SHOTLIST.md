## 1. 视觉总纲与连续性

- 场景依次覆盖中国大学男生宿舍 1107、十一楼走廊、消防楼梯间与不存在的四楼入口。红色马甲先以正常、温柔、近似可信的查房者出现，再在第一章内完成眼部血丝、面部错位与四肢拉长，并追赶三人至楼梯间。
- 画面母版为 `2160 x 3840`、`9:16`。关键脸、手、门锁、群人数与黑色头像均要同时落入手机安全区和桌面提取框。
- 1107 为四人间，上床下桌。主视角面对门时：铁门在画面右上，林峰床位在左前上铺，周朝阳床位在左后，谷雨床位在右后，宋明床位最靠近门且离开后保持空置；中央为下床后的通行区。除主观镜头外不得镜像翻转。
- 林峰固定穿洗旧灰色短袖与黑色运动短裤；周朝阳穿低饱和橄榄色上衣；谷雨穿褪色姜黄短袖并使用灰白格被；宋明穿米白短袖、深色短裤，肩搭青绿色条纹毛巾。章内不得换装。
- 光色以手机冷光、走廊偏绿荧光、微弱暖色桌灯构成三色关系。红马甲是查房与追逐段唯一高饱和红色信号；楼梯间转为病白应急灯与绿灰墙面，四楼入口只用失真的楼层灯和门牌反光，禁止整章持续红滤镜。
- 宋明的“真人状态”在第一章只允许背影、肩部、手或被门框截断的侧后轮廓。门外来者不得给清晰正脸，不能用视觉提前确认其真假。
- 禁止女性学生、女性寝具装饰、旧门牌、旧规则版式、旧人物剪影和任何旧女生宿舍素材进入本表资产。

1. 每个特效实例必须绑定 `shotId + FX ID + node token`。同一节点重复渲染先销毁旧实例，禁止叠加相机位移、滤镜、定时器或临时图层。
2. 快速点击采用“先结算稳定终帧，再推进节点”。点击期间不得排队重播震屏、敲门、黑屏、失焦或文字错位。
3. 节点切换、读档、切换故事、返回书架和销毁舞台时，统一取消 WAAPI、`requestAnimationFrame`、定时器、监听器、CSS 类、内联变换、临时 DOM/Canvas 和手机屏幕瞬态层。
4. `visibilitychange=hidden` 时暂停或销毁瞬态运动并清除相机变换；恢复前核对节点令牌。节点未变则显示稳定终帧，节点已变则完整清理。关门、敲门、闪黑、红马甲异化、走廊追逐、空间失焦和章末拉焦在恢复时均不重播。
5. 剧情持久结果与瞬态特效分离。门上最终划痕、消息失败、群禁言、黑头像占位、红马甲当前异化阶段、数字 4 和章末门牌可以恢复；震动、跑动位移、失焦、拖影、曝光闪断和临时影子不得恢复。
6. 开启降低动态后，不取消剧情信息，只把位移、缩放、震屏、呼吸失焦和拖影替换为 `80-180ms` 交叉淡变或静态终帧。

### 8.2 移动端遮挡规则

- 常规对白最多占底部 `34%`；多选项最多占底部 `48%`。角色眼睛、手与唯一线索不得进入对应遮挡区。
- 手机屏幕中的八条规则可独立滚动，外层剧情页面在阅读时锁定滚动；退出手机层后恢复原节点滚动位置。
- 黑色头像、群人数、失败标识、`01:00`、猫眼内红马甲、门上划痕、异化眼部/面部、消防门把、数字 `4`、姓名缺字和“三楼宿管宿舍”门牌不得被角色名牌、对话框、跑动区、选择按钮、返回按钮或底部手势区遮住。
- 多人镜头优先“一个清晰主脸 + 两个可读反应”，不把三张脸横向压成窄条；选择出现时三人整体上移，但脚部和床沿允许被遮挡。
- 震屏、跑动跟随、空间失焦和拉焦只作用于视觉舞台的指定分层，界面按钮、正文、手机滚动容器、跑动触控区和系统安全区保持稳定，避免误触与晕动。
- 最小可读文字按 `360 CSS px` 宽屏校验：规则正文不小于 `16px`，群人数和错误标识不小于 `14px`；不得把关键文字烘焙进低分辨率背景。
- 桌面裁切前先验证 `D-C/D-U/D-L/D-T/D-E`，手机再验证 `M-DIA/M-CHO/M-PHN/M-RUN/M-SPC/M-END`。任一端丢失唯一线索时必须重构图，不允许靠额外说明文字补救。
