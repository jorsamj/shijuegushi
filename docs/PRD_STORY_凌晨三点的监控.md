# 《凌晨三点的监控》故事剧情 PRD

## 1. 故事定位

《凌晨三点的监控》是“凌晨三点系列”的第一本剧本，用于建立系列的核心气质：现代智能系统介入古老空间，技术证据与人性秘密互相污染，所谓灵异现象最终指向被精心伪装的谋杀。

本 PRD 仅描述故事设定、人物、章节、线索、推理链、选择分支和结局，不展开完整对白脚本。

## 2. 故事类型

| 类型 | 说明 |
| --- | --- |
| 主类型 | 悬疑推理视觉小说 |
| 副类型 | 密室案件、技术诡计、旧案复仇 |
| 叙事视角 | 玩家扮演主角林澈，以第一现场参与者视角推进调查 |
| 推理方式 | 监控证据、现场痕迹、证词矛盾、系统日志交叉验证 |

## 3. 核心卖点

1. 古寺与智能监控系统的反差感。
2. “丢失的七秒”作为贯穿全剧本的核心谜题。
3. 密室死亡与灵异传闻互相掩护。
4. AI 预警、后台账号、预录画面等技术线索参与推理。
5. 四结局设计体现玩家推理完整度与道德选择。

## 4. 一句话故事概念

古寺智能监控上线前夜，财务负责人死在反锁钟楼中，监控丢失关键七秒；玩家必须证明这不是灵异事件，也不是系统故障，而是一场利用技术和旧案设计的谋杀。

## 5. 世界观与背景设定

故事发生在一座正在进行文旅数字化改造的古寺。寺内准备上线一套智能物联系统，包括监控、广播、门禁、夜间 AI 预警和后台日志。项目由林澈负责，许然代表政府侧协助验收。

古寺表面上是安静的文化场所，内部却长期存在财务旧账、香火资金去向和七年前事故传闻。智能系统上线前夜，第三路监控拍到未知人影，随后财务负责人周启山死于反锁的钟楼。寺中流传“凌晨三点有人敲钟”的旧传闻，凶手借此把谋杀伪装成灵异事件。

| 沈曼 | 死者侄女 | 查清周启山死亡原因 | 与七年前旧案存在间接关系，知道死者近期调查旧账 |

## 7. 人物关系

1. 林澈与唐越：项目负责人和现场工程师，存在责任链关系。
2. 林澈与许然：调查协作关系，许然提供行政侧信息和压力。
3. 周启山与慧明法师：寺内财务与住持关系，围绕旧账存在长期分歧。
4. 周启山与沈曼：叔侄关系，沈曼知道周启山近期异常紧张。
5. 陈伯与寺院众人：守夜多年，掌握寺内日常异常。
6. 凶手与周启山：因七年前旧账和即将曝光的证据产生直接冲突。

## 9. 案件真相

周启山并非死于灵异事件，也不是智能系统误判导致的事故。他因发现七年前古寺修缮旧账中的关键证据，被凶手设计诱至钟楼。凶手利用后台账号、广播系统和预录监控制造时间差，掩盖自己进入和离开现场的过程，并让“第三路监控丢失 7 秒”看起来像系统故障。

钟楼密室并非真正密室。门锁内侧划痕、水痕和广播启动记录共同证明，现场被人为处理过，反锁状态是事后制造的。

## 10. 凶手动机

凶手的核心动机是阻止周启山公开七年前旧账真相。旧账一旦公开，将牵连寺院资金、文旅项目验收和旧案责任人。凶手不是为了单纯杀人，而是为了让周启山的死亡被归因于古寺传闻或系统故障，从而同时压下旧案和新项目风险。

注：具体凶手身份可在开发前最终确认。建议第一版将凶手设定为掌握寺内空间和旧账信息的人，并通过结局设计允许玩家误判唐越或系统。

## 11. 作案手法

1. 利用匿名信息诱导周启山在凌晨前往钟楼。
2. 借用或盗用唐越后台账号，制造工程师嫌疑。
3. 在后殿监控中插入预录画面，伪造某段时间的走廊状态。
4. 在 03:17 启动广播系统，用声音掩盖争执或机关声。
5. 制造第三路监控 7 秒缺失，隐藏关键移动过程。
6. 处理钟楼门锁，使其呈现内侧反锁假象。
7. 利用地面水痕破坏脚印和拖拽痕迹。
8. 借古寺凌晨传闻引导众人往灵异方向联想。

## 12. 核心谜题

1. 第三路监控为什么正好丢失 7 秒？
2. 周启山为什么没有被拍到进入钟楼？
3. 反锁钟楼如何成为伪密室？
4. 唐越账号登录是否代表唐越本人作案？
5. 广播系统为何在 03:17 启动？
6. 后殿监控中的人影是真实画面还是预录画面？
7. 七年前旧账与当晚死亡有什么关系？

## 20. 分支选择设计

分支不需要在第一版做成完全不同的长线剧情，建议采用“线索完整度 + 关键判断”的方式控制结局。

分支变量建议：

| 变量 | 含义 |
| --- | --- |
| found_lost_7_seconds | 是否发现 7 秒缺失 |
| found_lock_scratch | 是否发现门锁划痕 |
| found_water_trace | 是否发现水痕 |
| found_broadcast_log | 是否发现广播启动记录 |

| found_prerecording | 是否发现预录痕迹 |

## 21. 玩家推理流程

1. 发现监控异常：第三路监控丢失 7 秒。
2. 发现现场异常：反锁状态和水痕不自然。
3. 发现技术嫌疑：唐越账号登录后台。
4. 发现技术嫌疑不充分：账号可被借用或盗用。
5. 发现旧案动机：周启山正在调查七年前旧账。
6. 发现监控被替换：后殿监控存在预录痕迹。
7. 串联作案链：广播掩盖声音，7 秒遮蔽移动，预录画面制造不在场证明。
8. 最终判断：这是一场借系统故障和古寺传闻伪装的谋杀。

## 22. 最终推理题设计

最终推理建议使用 4 个问题，每题对应一个真相环节。

| 题目 | 正确方向 |
| --- | --- |
| 监控最关键的异常是什么？ | 第三路监控丢失 7 秒，且后殿画面存在预录痕迹 |
| 钟楼为什么不是真正密室？ | 门锁划痕和水痕说明现场被人为处理 |
| 唐越账号登录说明什么？ | 说明后台被使用，但不能单独证明唐越作案 |
| 周启山为什么会死？ | 他调查七年前旧账，威胁到旧案相关人 |

玩家需要在最终推理中选择证据组合。若关键证据不足或选择错误，则进入坏结局。

## 23. 结局触发条件

| 结局 | 名称 | 触发条件 |
| --- | --- | --- |
| A 结局 | 真相公开 | 收集关键线索不少于 6 条，发现旧账和预录痕迹，最终选择公开真相 |
| B 结局 | 抓错凶手 | 过早怀疑唐越，缺少预录痕迹或旧账线索，最终指认唐越 |
| C 结局 | 系统背锅 | 收集系统异常线索但缺少旧账动机，最终将事件归因于系统故障 |
| D 结局 | 沉默的古寺 | 已接近真相但最终选择沉默，或关键角色施压后不公开证据 |

## 24. 好结局与坏结局差异

| 维度 | A 结局 | B/C/D 结局 |
| --- | --- | --- |
| 真相完整度 | 完整揭示旧账、作案手法和伪密室 | 只解释部分异常或回避真相 |
| 角色命运 | 周启山死亡真相被公开，唐越洗清主要嫌疑 | 唐越、系统或寺院传闻承担错误后果 |
| 系列延展 | 引出更深的凌晨三点系列线索 | 留下更强烈的不安和未解问题 |
| 玩家反馈 | 明确奖励推理完整度 | 强调证据不足和选择代价 |

## 25. 剧情节奏设计

1. 开场 10%：用第三路监控未知人影和死亡事件抓住注意力。
2. 前段 30%：围绕钟楼现场建立密室疑问。
3. 中段 30%：通过后台日志和唐越账号制造技术嫌疑。
4. 后段 20%：旧账线索出现，案件从系统异常升级为人性动机。
5. 结尾 10%：玩家整合线索，完成最终推理并进入结局。

节奏原则：每章至少抛出一个新问题，同时回答前一章的部分疑问，避免线索堆积但没有推进。

## 26. 悬疑氛围设计

1. 古寺夜间环境：钟楼、后殿、雨水、回声、木门和监控死角。
2. 技术不可靠感：日志缺失、预警误报、后台账号异常。
3. 灵异误导：凌晨三点人影、无人钟声、反锁钟楼。
4. 现实落点：所有灵异感最终都能被技术诡计和人类动机解释。
5. 文本风格：克制、冷静、留白，不依赖夸张惊吓。

## 27. 第二本《凌晨三点的来电》的伏笔

1. A 结局后，林澈收到一通没有来电号码的电话，对方只说“七秒不是第一次”。
2. 唐越提到系统曾在另一处项目中记录过类似的凌晨异常。
3. 许然在旧账附件里发现一个被涂黑的电话号码。
4. 沈曼暗示周启山死前曾反复接到凌晨三点的来电。
5. 后殿预录文件的命名规则中出现未知前缀，可作为第二本的线索入口。
