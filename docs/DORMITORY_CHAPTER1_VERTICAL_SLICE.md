# 《宿舍规则怪谈》第一章移动端交互纵切

> 章节名：《不要忘记你的名字》
>
> 范围：仅本次第一章纵切；从午夜陌生短信开始，经过红色马甲查房、异化与逃离，在三楼宿管宿舍入口结束。
>
> 正典边界：发生在中国大学男生宿舍。主控为林峰，队友为周朝阳、谷雨、宋明；不使用旧版女生宿舍人物、房号、规则、线索、视觉或语音。

## 1. 纵切目标

这一章要让玩家亲手得到四个结论：规则会造成真实后果；班群的黑色头像并非新增成员；林峰的身份已经开始被挤出记录；红色马甲查房者不是宿管。章节不解决宋明的真假，也不进入四楼；结尾只抵达三楼宿管宿舍的半掩入口。

- **有效选择：12 个。** 每个都至少改变一项可回收内容：证据是否保留、规则判断、林峰被暴露的程度、周朝阳或谷雨的信任、宋明相关线索，或后续章节的对话与风险。
- **限时互动：2 次。** 关门拖拽（4 秒）与红色马甲的应答判断（5 秒）。超时不会强制读档，而是进入带代价的剧情降级。
- **手机实际互动：3 次。** 陌生短信、班群规则/成员页、给宋明发消息均在可操作手机模式中完成，不替换成普通选项按钮列表。
- **规则查看/对照：2 次。** 首次完整阅读八条学生规则；门外敲门时对照“不要开门”“午夜后不得在寝室外逗留”“一点至三点查房”。
- **恐怖演出锚点：2 个。** 门缝瘦长影子与多方向宋明声音/划门；均有明确起止与降动态方案。

## 2. 移动端表现约定

### 手机模式

手机以全屏内嵌设备呈现：可见锁屏通知、状态栏、消息会话、班群、成员抽屉、输入框、返回手势和截图预览。剧情层在手机打开时冻结，但环境底噪与世界内来电/敲门仍可打断手机；玩家返回后回到同一滚动位置。

短信处理、班群阅读、规则标记和给宋明发消息都通过点击会话、长按消息、上滑成员页、拖动阅读进度、输入短句或发送图标完成。不能把它们做成“保存短信 / 查看规则 / 给宋明发消息”的普通横向按钮列表。手机内没有“确定”“我知道了”一类高频确认弹窗；成功、失败和状态都由消息形态、撤回、无法送达、联系人缺字、角色行为及后续对白表达。

### 可见反馈与隐式连续性

玩家只看到中文人名、文本缺字、群成员变动、照片/头像异常、角色站位和剧情性存档标题。不得显示信任值、污染值、正确答案、技术枚举或调试状态。设计记录中的“信任”“暴露”“证据”只是实现与 QA 用的隐式连续性，绝不出现在玩家界面。

### 存档与演出总则

- 章节设置四个自动检查点：短信处理后、门外宋明首次敲门前、红色马甲异化后、抵达三楼宿管宿舍入口时。手动存档只在静止叙事段可用，不在两个倒计时内写入半完成动作。
- 读档恢复至最近完整事件边界；已完成的手机操作、规则标记和角色反应以当时的画面形态重建，不重复播放已经结算的突脸、震动或惊吓音。
- 节点离开、读档、切换故事或返回书架时，取消倒计时、震动、屏幕抖动、失焦、闪灯、音频淡入淡出和延迟回调；降低动态效果时，连续抖动改为一次静帧错位，频闪改为暗场切换，仍保留关键信息。
- 旁白、动作、环境与系统说明保持静音。只有角色说出口的话、手机通知/录音、敲门、划门和门外查房声可以发声。

## 4. 互动合同：输入、防误触、失败降级、存档与生命周期
