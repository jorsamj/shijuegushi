## 3. 一句话故事概念

暴雨夜，林舟接到三年前已故室友许知夏的电话：“别开门，她不是我。”几秒后，一个和许知夏极像的女人站在门外，自称许知夏的妹妹。

## 4. 故事简介

林舟独自在出租屋加班，突然接到来自已故室友许知夏旧号码的电话。电话只留下一句警告，门铃便响起。门外的许知晚声称自己是许知夏的妹妹，并带来旧手机、旧案疑点和一张可能藏着关键证据的大学合照。

玩家需要在信任、恐惧、旧友压力和现实证据之间做出选择，判断门外女人的身份，查明许知夏三年前死亡真相，并决定是否重启旧案。

## 5. 新叙事节奏原则

### 5.1 更长沉浸段落

1. 减少频繁章节切换。
2. 每章形成一个完整悬疑段落。
3. 玩家连续经历一段危机、调查或反转。
4. 章节结算只在 6 个大章节结束时出现。

### 5.2 更强悬疑钩子

1. 每章开头有明确悬念。
2. 每章中段有压力升级。
3. 每章结尾有强钩子。
4. 不要只是交代信息，要让玩家有继续点击的冲动。

### 5.3 更少但更关键的线索

1. 每条正式线索必须影响最终推理或结局路径。
2. 不把所有信息都做成线索。
3. 让玩家记得住线索，而不是被线索淹没。

### 5.4 更强人物压迫

1. 许知晚可信但不透明。
2. 周屿前期像关心旧友，后期逐步失控。
3. 陈妍提供现实支撑，但不要替玩家推理完。
4. 林舟有逃避、愧疚和被迫面对的心理线。

### 5.5 旁白少于对话和动作

1. 旁白用于气氛和衔接。
2. 对话用于试探、冲突和信息交换。
3. 动作用于制造危机、选择压力和真实感。
4. 旁白不超过 35%，对话、动作、当前冲突和选择内容至少 65%。

## 7. 案件真相

许知夏三年前不是意外死亡。她发现周屿冒用她的信息进行灰色借贷，并准备报警。周屿为了阻止她揭发，制造坠楼意外。

许知晚这些年一直在查姐姐的死。她来找林舟，是因为林舟手里可能有最后一张大学合照。合照背景中拍到了周屿在案发楼道附近出现，与他的证词矛盾。

所谓“死者来电”不是灵异，而是许知夏旧手机被许知晚重新开机后，旧语音备忘或云端记录触发。周屿不是制造原始来电的人，而是顺势利用这通电话制造恐惧和误导。

## 8. 新 6 章结构

### 第 1 章：雨夜来电

内容：

1. 林舟加班、出租屋生活细节。
2. 死者来电。
3. “别开门，她不是我。”
4. 门铃响起。
5. 猫眼外女人出现。
6. 林舟第一次选择：不开门、询问、报警、联系旧友。

本章目标：开场强钩子，不要铺垫太久，但要有生活真实感。

本章结尾钩子：门外女人说出只有许知夏知道的旧事，同时周屿发来消息：别让她进去。

### 第 2 章：门外的许知晚

内容：

1. 许知晚隔门解释身份。
2. 林舟核验她的细节。
3. 许知晚拿出旧手机照片，但旧手机本身不作为正式线索。
4. 周屿继续发消息干扰。
5. 许知晚隐瞒她如何找到林舟。
6. 林舟在相信和怀疑之间摇摆。

本章目标：围绕“她到底是谁”制造身份疑云。

本章结尾钩子：许知晚发来的旧手机截图里，屏幕上出现林舟的名字。

### 第 3 章：三年前的裂缝

内容：

1. 旧聊天记录和大学回忆，但不作为正式线索。
2. 周屿当年的控制欲。
3. 陈妍帮助查旧新闻和灰色借贷。
4. 许知夏死亡前搜索“身份信息被冒用怎么办”。
5. 周屿案发后提前离城。
6. 林舟意识到这不是意外。

本章目标：把旧案从“怪事”推进到“有人害她”。

本章结尾钩子：陈妍发现周屿离职时间早于新闻报道。

### 第 4 章：最后一张合照

内容：

1. 林舟翻找旧物。
2. 找到最后合照，但合照本身不作为正式线索。
3. 合照背面有模糊字迹，但不作为正式线索。
4. 照片放大检查。
5. 背景里出现周屿身影。
6. 房东老太提供楼道证词，但不作为正式线索。
7. 周屿开始直接施压。

本章目标：让玩家真正参与“照片推理”。

本章结尾钩子：周屿说出林舟刚刚找到合照这件事。

### 第 5 章：旧手机的声音

内容：

1. 许知晚解释旧手机开机与云端恢复。
2. 林舟和陈妍判断死者来电机制。
3. 旧语音备忘播放。
4. 录音中出现“周屿他……”。
5. 周屿电话正式接入。
6. 楼道灯灭。
7. 门外出现第二个人脚步声。

本章目标：解释伪灵异机制，同时把现实威胁推到最高。

本章结尾钩子：周屿在电话里问：“你听到录音了？”

### 第 6 章：无人接听

内容：

1. 林舟最终整理证据。
2. 人物关系和选择压力集中。
3. 最终推理 5 题。
4. 玩家选择备份、交出、删除或逃避。
5. 自动进入 A/B/C/D 结局。
6. 结局报告展示路径链。

本章目标：完成推理、选择和结局复盘。

## 9. 正式线索压缩

正式线索只保留 6 条：

| clueId | 名称 | 作用 |
| --- | --- | --- |
| clue_dead_call | 死者来电 | 证明雨夜电话发生。 |
| clue_sister_mark | 许知晚身份细节 | 支撑许知晚身份可信但不完全透明。 |
| clue_gray_loan | 灰色借贷 | 指向周屿作案动机。 |
| clue_zhou_left | 周屿离城 | 指向周屿时间线异常。 |
| clue_photo_background | 照片背景人影 | 关键物证，推翻周屿证词。 |
| clue_timed_voice | 旧手机语音触发记录 | 解释死者来电机制。 |

删除正式线索：`clue_warning_sentence`、`clue_door_woman`、`clue_old_phone`、`clue_last_photo`。

辅助信息不进入线索库：房东证词、旧聊天记录、周屿异常短信、合照背面字迹、旧手机开机时间。这些只能作为剧情证据细节、正文信息、结局路径解释，不弹线索卡。

## 10. 选择节点要求

每章至少 3-5 个有效选择。每个选择要有后果。

关键选择至少覆盖：

1. 是否开门。
2. 是否要求许知晚证明身份。
3. 是否给陈妍发消息。
4. 是否相信许知晚。
5. 是否接周屿电话。
6. 是否质问周屿。
7. 是否备份照片。
8. 是否把原始照片交给许知晚。
9. 是否删除证据。
10. 是否选择重启旧案。
11. 最终推理选择。
12. 是否逃避调查。

每个关键选择要说明影响哪些人物关系值、flags、结局倾向，以及是否进入结局路径说明。

## 12. 角色对白规范

### 林舟

不是工具人。她有疲惫、逃避、愧疚、迟疑。越接近真相，越主动。

### 许知晚

可信但不完全透明。说话急，但不应一次性解释完真相。她越急，玩家越难判断。

### 周屿

前期语气像关心旧友；中期开始“替你着想”；后期逐渐暴露控制和威胁。他要说出一些不该知道的信息。

### 陈妍

不要像工具人。可以吐槽、怀疑、提醒。她提供现实支撑，但不替玩家做完推理。

### 房东老太

不作为大段解释角色。只提供生活化、模糊但有用的现场细节。

## 13. 悬疑钩子设计

每章每隔 5-8 个节点出现一个小悬念：

1. 周屿发来一条消息后又撤回。
2. 许知晚说错一个细节。
3. 门外出现第二个脚步声。
4. 旧手机自动亮屏。
5. 照片里有一个反光不对。
6. 陈妍查到的新闻时间和记忆不一致。

章末强钩子示例：

1. “别让她进去。”来自周屿。
2. 旧手机截图里出现林舟的名字。
3. 周屿离职时间早于新闻报道。
4. 照片背景里的人影，穿着周屿那件旧外套。
5. 周屿在电话里问：“你听到录音了？”
6. 最终选择前，门外传来钥匙插进锁孔的声音。

## 14. 四结局路径

### A 结局：真相重启

路径链：查到灰色借贷；发现周屿离城；发现照片背景人影；理解死者来电机制；备份证据；最终推理达到 4/5；选择重启旧案。

### B 结局：证据失控

路径链：找到关键证据；选择交出原始照片；身份核验、证据备份或来电机制存在缺口；证据链流转失控。

### C 结局：删除证据

路径链：选择删除证据；没有建立备份；关键证据消失。

### D 结局：无人接听

路径链：没有形成完整证据链，或没有选择重启旧案，或最终推理不足；真相停在无人接听的电话里。

## 15. 最终推理题

1. 死去三年的许知夏为什么会来电？
   正确方向：旧手机开机后，云端语音备忘触发提醒。
2. 门外的许知晚最可能是谁？
   正确方向：许知夏的妹妹，但隐瞒部分调查方式。
3. 三年前许知夏为什么会死？
   正确方向：她发现周屿冒用身份借贷，并准备报警。
4. 最后合照真正的价值是什么？
   正确方向：背景拍到周屿出现在案发楼道附近。
5. 周屿当前最想阻止什么？
   正确方向：阻止照片、录音和借贷资料形成完整证据链。
