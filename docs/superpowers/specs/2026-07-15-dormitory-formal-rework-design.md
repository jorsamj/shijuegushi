# 《宿舍规则怪谈》正式返工设计

## 目标

将第二故事从“可跑通的规则怪谈原型”升级为移动端优先的正式互动视觉小说：六章形成连续因果链，30 至 40 个有效选择持续改变信任、身份泄露、幸存者与路线；手机、广播、现实现场和八个结局共享同一时间线；画面、人物状态、手机内容和恐怖演出由节点数据共同驱动。

《雨夜来电》继续使用现有剧情、视觉和语音，只接受玩家可见中文、碎片文本框与共享运行时的回归修复。

## 叙事结构

- 第一章由规则广播触发床位危机，走廊伤亡导致“现实死亡、账号在线”的首个矛盾，并以 415 内外同时出现陈露收束。
- 第二章让玩家选择验证手段，误判真实造成伤亡，验证体系因此失去公信力，并以两个陈露账号收束。
- 第三章把威胁转入手机：视频、镜像、门牌、动态口令与沈妍用词习惯相互矛盾，最终把私人信息泄露给第四章的亲人模仿。
- 第四章让许棠母亲与林穗姐姐的模仿同时发生，玩家的安慰、隐瞒或怀疑长期改变两人信任，并引出互相冲突的两段广播。
- 第五章通过联系吴阿姨、比对提示音、旧群聊和值班记录逐步重建 2014 年事件，确定 01:13 的九十秒窗口、真假路线与逃生队伍。
- 第六章只保留连续逃生和即时判断，回收身份泄露、动态口令、真实人数、救援承诺、路线与信任，最后在闸机前进入八个正式结局。

八个结局继续为《真正的天亮》《她替我关上门》《被留下的人》《合法人数》《第二个许棠》《三人在线》《东侧安全通道》《砸碎广播》。每个结局必须拥有独立画面、手机终态、人物结果、最后一句、前期选择回收和至少两处可检查的伏笔。

## 节点与运行时合同

第二故事节点在保留现有字段的基础上增加：

- `visualCast`: 多人物图层数组，成员包含 `characterId`、`variant`、`position`、`framing`、`mobileFraming`、`focusX`、`focusY` 与 `zIndex`。
- `phoneScreen`: 结构化手机界面，包含 `kind`、`time`、`battery`、`signal`、`title`、`members`、`messages`、`typing`、`call`、`video` 与 `systemNotice`；玩家可见文本全部由 DOM 渲染，不烘焙进生成图片。
- `effects`: 事件特效数组，成员包含 `type`、`level`、`durationMs`、`target`、`delayMs` 和 `cleanup`。同一节点只允许一个重度效果，普通对话不得持续抖动。
- `transition`: 仅在空间或时间真正改变时使用；同场景节点默认保持背景，只替换人物、焦点、手机和短暂效果。
- `feedback`: `inline`、`toast`、`chapter-card` 或 `none`。第二故事的选择、线索、规则与身份变化不得进入强制确认模态框。
- `foreshadowingIds`、`consequenceIds`、`survivalUpdates` 与 `identityUpdates`: 为因果、反转和结局 QA 提供可追踪状态。

共享引擎保持向后兼容：旧节点仍由 `speaker`、`visualCharacter`、`effect` 与 `phoneState` 渲染；新字段仅在第二故事出现。场景切换、读档、返回书架和故事切换必须清理特效计时器、手机震动与旧语音实例。

## 视觉系统

- 建立同一栋中国大学女生宿舍的空间圣经：417、415、419 位于四楼同一走廊，东/西楼梯、一楼值班室、广播控制室和闸机具有可连续追踪的门窗与方向。
- 背景至少覆盖 417 三种灯光、415、419、四楼普通/红灯/猫眼、门缝血迹、楼梯与两侧通道、一楼值班室/广播室/闸机、楼外清晨和八个独立结局。
- 角色默认使用胸像、半身或面部近景；全身只用于走廊距离、双人出现、拖拽、追逐、死亡与结局。手机端人物视觉中心位于上半屏，人物脸、眼睛、嘴和关键手势不得落入底部对话框区域。
- 七名角色建立固定脸型、发型、服装、身材与年龄基准；每名主要角色只生成剧情必需状态。伪人异常由嘴唇不同步、眼神、影子、镜像、关节或局部错位表现，不使用统一红眼。
- 手机画面采用可测试的 HTML/CSS 组件，并使用头像与视频缩略图等少量图像资产；时间、电量、在线人数、消息顺序和昵称由节点数据保证一致。

## 交互与演出

- 选择、线索、规则更新和联系人状态使用非阻断 HUD 反馈；规则板、线索库、设置和存读档仍可使用功能面板。只有覆盖存档、删除、重开、退出等不可逆操作保留确认模态框。
- 三级演出统一由 `SceneEffectController` 管理：轻度包含手机震动、噪点和灯光变化；中度包含撞门、黑屏、信号撕裂、局部血迹和人物错位；重度包含拖拽、袭击、追逐、伪人暴露与坏结局。
- 所有效果必须在节点离开、读档、返回书架、页面隐藏或完成后清理。快速点击不得叠加动画。
- 新增“降低动态效果”设置，并同时遵从 `prefers-reduced-motion`。降低后保留剧情信息，但把抖动、闪烁和快速位移替换为短暂暗角、亮度渐变或低幅模糊。

## 发布门槛

- 自动验证六章、30 至 40 选择、八结局、因果引用、伏笔、中文可见名、碎片文本、手机时间线、资产路径、角色状态、空间连续性、弹窗预算、特效生命周期与第一故事回归。
- 桌面与移动端至少覆盖 360x640、375x667、390x844、412x915、768x1024 和 1440x900；检查人物遮挡、选择首屏、对话框、手机层、特效清理和降低动态效果。
- 第二故事新文本稳定前保持正式配音运行时为空；完成文本审计后只为变化或新增的真实发声节点生成 staging 音频，再通过完整性校验原子切换。
- PR #6 始终保持 Draft，不合并 `main`，不发布 Pages，不转为 Ready。
