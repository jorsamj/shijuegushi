# 《宿舍规则怪谈》正式返工 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 重建第二故事的六章因果叙事、统一视觉世界、移动端镜头、手机界面与三级恐怖演出，并保持第一故事兼容。

**Architecture:** 使用数据驱动的第二故事节点合同扩展共享静态前端引擎。场景、多人镜头、手机界面、非阻断反馈和可清理特效彼此独立，通过节点数据在同一渲染周期组合；旧故事字段继续兼容。

**Tech Stack:** Vanilla JavaScript、HTML、CSS、Web Audio、localStorage、Node.js QA scripts、PNG/WebP 图像资产。

---

### Task 1: 建立返工门禁

- [ ] 新增失败的剧情、中文名、碎片文本、手机时间线、视觉资产、弹窗预算和特效生命周期检查。
- [ ] 运行新增检查并确认它们因当前数据缺失而失败。
- [ ] 将新增检查接入 `check-production-readiness.mjs`，保持人工签核门禁不变。

### Task 2: 重写六章数据与八结局

- [ ] 按设计文档重写第二故事节点，保持六章、30 至 40 个有效选择和八个结局。
- [ ] 为每个主要事件添加明确的 `foreshadowingIds`、`consequenceIds`、幸存者、身份、手机和人物关系更新。
- [ ] 清理无意义短旁白，保持人物对白纯净，确保纯旁白静音。
- [ ] 更新结局解析与八条自动路线，使错误路线不能误入《真正的天亮》。

### Task 3: 建立统一视觉资产

- [ ] 编写角色与空间视觉圣经、镜头表和资产命名表。
- [ ] 生成并人工检查七名角色的主视觉与必要状态，确保跨图身份、服装和年龄一致。
- [ ] 生成并检查宿舍、走廊、楼梯、一楼和八结局背景，确保空间方向与结构一致。
- [ ] 生成手机头像、视频缩略图、异常局部和血迹等必要前景资产；手机文字由 DOM 渲染。
- [ ] 更新资产 manifest 和映射，旧资产移入非运行时归档或不再引用。

### Task 4: 实现多人镜头与移动端构图

- [ ] 为共享引擎增加 `visualCast` 兼容层并编写失败测试。
- [ ] 实现桌面/手机差异化 framing、焦点与安全区，保留旧单人物节点兼容。
- [ ] 预加载当前与下一节点的背景、人物和手机缩略图，加载失败保持上一帧并显示克制状态。
- [ ] 在目标手机尺寸逐节点检查重点场景和八结局的面部遮挡。

### Task 5: 实现手机叙事与非阻断反馈

- [ ] 为 `phoneScreen` 编写结构校验和失败测试。
- [ ] 实现群聊、私聊、语音、图片、视频、来电、系统通知和在线状态组件。
- [ ] 将第二故事选择、线索、规则、身份与章节反馈改为 HUD/内嵌反馈；仅不可逆操作保留模态确认。
- [ ] 验证快速点击、选择锁、手机滚动和功能面板不会误推进剧情。

### Task 6: 实现三级恐怖演出与降低动态效果

- [ ] 为效果队列、单重度限制和清理生命周期编写失败测试。
- [ ] 实现手机震动、撞门、黑屏、红灯、视频撕裂、人物错位、血迹、拖拽、追逐、最终冻结和坏结局效果。
- [ ] 节点离开、读档、故事切换、页面隐藏和效果完成时统一清理。
- [ ] 新增持久化降低动态效果设置，并提供 `prefers-reduced-motion` 自动降级。

### Task 7: 回归、视觉验收与发布分支收口

- [ ] 运行所有剧情、资产、音频、生命周期、多故事和 production readiness 自动检查。
- [ ] 用桌面和六种移动视口跑通六章与八结局，检查中文、遮挡、弹窗、手机时间线和特效清理。
- [ ] 完整回归《雨夜来电》，确认剧情、图片、语音、存档和四结局不受影响。
- [ ] 记录仍需人工试听/审美签核的内容，提交并推送当前 `codex/dormitory-rules-story` 分支；保持 PR #6 为 Draft。
