# 《宿舍规则怪谈》分支与结局矩阵

> 文档性质：第二故事剧情设计稿，只服从 `docs/DORMITORY_STORY_BIBLE.md`。
>
> 范围：七章有效选择、延迟回收、人物死亡续写、姓名污染、信任、线索、场景与八结局判定。
>
> 正典隔离：本矩阵仅使用林峰、周朝阳、谷雨、宋明、吴阿姨、男生宿舍、十一楼、三楼宿管宿舍、无名四楼与校长室；不引用任何旧版废弃基线的角色、门牌、规则、线索、分支或结局条件。
>
> 章节边界特批：按用户本轮明确要求，第一章延长至红马甲破门追逐并抵达数字“4”；这只替代剧情圣经第 10 节中第一、二章的切章位置，不改变事件顺序、规则、人物关系或七章总数。

## 1. 使用约定

### 1.1 选择有效性

- 全文共 **55 个选择点**，其中 **17 个限时选择**。限时选择以 `【限时 N 秒】` 标记，超时结果写在选项末尾，不使用随机结果。
- 每个选项至少改变以下两类中的两类：人物状态、姓名污染、信任、证据、场景路径、羽毛、名册或结局资格；不存在只改下一句对白的空选择。
- “已知 / 未知”只描述玩家在当时能合理掌握的内容。身份真相、延迟死亡和结局资格不会被即时系统卡片宣布。
- 内部状态可用于编剧和实现校验，但玩家端只显示缺字、联系人变化、群成员替换、合照模糊、称呼改变、存档名异常等世界内反馈，不显示裸数值。

### 1.3 隐藏事实的确定规则

1. 开场门外的第一个“宋明”是尚未完整复制身份的模仿者；真正的宋明被拖入复制 1107。
2. 若“宋明全名泄露”（C04-H 或 C19-C/D）且“救援迟到”（C20-C/超时或 C23-B/E）同时成立，真正宋明在第四章前死亡；之后同行者是继承其记忆、并可按第 4 条取得自我资格的模仿者。否则，C21 的有效验真可以救出真正宋明。
3. 玩家控制的林峰起点为真人。若 C11-D 或 C18 超时造成身体中断，且当时既无羽毛护持也无活人姓名锚定，则置为“玩家林峰中途被替”；第七章拥有合法身份的另一个林峰是被困的原身。若未触发替换，另一个林峰由四楼根据记录复制。名册毁坏还可能使两者都失去“最初者”的可证明性。
4. “伪人产生自我”必须同时拥有一段未被设备记录的新记忆和一次自愿承担风险的行为，不能仅靠旧记忆、学籍或账号判定。
5. 所有超时结果固定写明；同一存档、同一状态、同一选择必得同一后果。
