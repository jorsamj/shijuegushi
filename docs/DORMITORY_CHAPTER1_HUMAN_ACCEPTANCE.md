## 2. 实际游玩顺序

1. 午夜 00:00，宋明披毛巾离开 1107，林峰收到“不要忘记你的名字”。
2. 玩家保存、回复、拉黑、转发或保留短信；选择写入后续状态，不出现正误提示。
3. 门未关严，瘦长影子经过；玩家限时关门、叫住宋明或僵在原地。
4. 周朝阳和谷雨醒来，三人确认宋明不在床上。
5. 黑色头像在禁言班群一次性发布完整八条规则；玩家查看规则、群人数与成员。
6. 门外宋明敲门；玩家阻止谷雨、打开私聊、决定是否发消息并查看猫眼。
7. 玩家选择沉默、无声回应、询问未记录事件或错误触碰门把，再在手机规则记录中作出判断。
8. 隔壁因门外对象答出生活细节而开门；玩家选择警告、沉默或录音，随后发生撞击、拖拽和血迹演出。
9. 班群短暂显示宋明在线，林峰的群名变成“林?”；时间推进到 01:00。
10. 红色马甲以温柔查房声诱导回应；玩家对照规则并限时决定沉默、询问、报人数或开门。
11. 红色马甲逐渐发生面部错位与四肢拉长；三人冲入楼梯间并限时选择逃跑方式。
12. 楼层数字从 11、3 变成 4；周朝阳接近观察，林峰与谷雨阻止。
13. 周朝阳第一次询问林峰姓名；回答发生卡顿，四楼走廊复制十一楼结构。
14. 走廊尽头出现“三楼宿管宿舍”门牌，第一章在规则矛盾处结束并返回书架。

## 4. 按流程人工检查

### 4.1 开场与陌生短信

- [ ] 陌生短信在开场两次自然推进内出现，悬念建立足够快。
- [ ] “不要忘记你的名字”在手机中清楚醒目。
- [ ] 保存、回复、拉黑、转发和保留的含义无需教程即可理解。
- [ ] 宋明离开宿舍有动作、对白和门未关严的连续过程。
- [ ] 选择后没有“正确”“错误”或信任数值弹窗。

### 4.2 班群与八条规则

- [ ] 黑色头像异常明确，但没有提前解释真相。
- [ ] 群人数仍为 42 与成员中出现黑色头像的矛盾清楚。
- [ ] 八条学生规则在同一手机页面完整出现，没有拆散播报。
- [ ] 规则页可以滚动、返回和再次从工具栏查看。
- [ ] 手机状态栏、时间、信号、电量、群标题与人数一致。
- [ ] 无乱码、英文人物名、`nodeId`、`roleId`、`basic-credible` 或其他内部枚举。

### 4.3 门外宋明

- [ ] 敲门前有安静、关门和三人反应的铺垫。
- [ ] 三次敲门、私聊、猫眼和规则对照形成连续验证过程。
- [ ] “不发送”不会伪造一条“发送失败”的消息。
- [ ] 周朝阳阻止开门符合逻辑型人物设定。
- [ ] 谷雨想回应的犹豫自然，不是单纯推动按钮。
- [ ] 错误触碰门把或喊全名会留下暴露状态，但不立即判死。

### 4.4 隔壁宿舍遇袭

- [ ] 隔壁先盘问床位、门卡和球鞋，开门有明确原因。
- [ ] 玩家可以警告、保持安静或录音；选择写入状态。
- [ ] 瞬时黑屏、重物撞击、拖拽和血迹依次出现。
- [ ] 谷雨捂嘴、林峰反胃、周朝阳挡在前方，袭击后有人物反应。
- [ ] 血迹不会长期遮挡文字和选项。

### 4.5 红色马甲查房

- [ ] 手机与电子钟明确推进到 01:00，符合查房规则。
- [ ] 门外声音显示为“门外女声/红色马甲宿管”，不会误标成“宿舍广播”。
- [ ] 玩家能在工具栏重新查看绿色马甲规则。
- [ ] 正常伪装先出现，面部错位和四肢拉长随后发生。
- [ ] 沉默、询问、报人数和开门分别留下不同暴露状态。
- [ ] 追逐方向选择适合单手操作且不被底部安全区遮挡。

### 4.6 不存在的四楼

- [ ] 宿舍门、消防门、楼梯和楼层数字形成可理解的逃跑路线。
- [ ] 数字 11、3、4 的变化明确回收“学校没有四楼”。
- [ ] 周朝阳先靠近观察，林峰再发火，人物动机顺序完整。
- [ ] 姓名异常只在四楼钩子明确发生，不提前重复解释。
- [ ] “三楼宿管宿舍”门牌出现在四楼走廊，能引发继续游玩的欲望。
