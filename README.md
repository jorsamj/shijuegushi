《雨夜来电》

## 当前文档

- [docs/STORY_SCRIPT_雨夜来电.md](docs/STORY_SCRIPT_雨夜来电.md)：《雨夜来电》剧情脚本文档，采用 6 个长章节结构，描述章节、节点、选项、线索、变量和结局节点。

## 本轮重构说明

- 将 12 个短章节调整为 6 个长章节。
- 将正式线索压缩为 6 条关键线索。
- 减少剧情过程中的强制弹窗。
- 强化悬疑张力、连续阅读体验和有效选择。
- 优化未解锁内容展示，避免大量问号占位。
- 降低旁白比例，提高对话、动作和当前冲突比例。

## 本轮视觉增强

- 使用图像生成方式制作可复用素材包，并裁切落地为 WebP / PNG 文件。
- 新增背景、叠加层、角色、道具、线索、章节封面、结局图和 UI / FX 素材目录。
- 游戏页按背景层、环境叠加层、道具层、人物层和 UI 层组合画面。
- 线索库、人物关系面板、章节结算和结局报告接入真实图像素材。
- 不依赖外链图片，后续可逐步替换或扩展单个素材。

## 运行说明

在仓库根目录启动本地静态服务：

```bash
python -m http.server 8080
```

然后访问：

```text
http://localhost:8080
```

## 公开部署

仓库内置 GitHub Pages 工作流：`main` 分支更新后会发布静态站点。首次发布前，请在仓库 Settings -> Pages 中将 Source 设为 **GitHub Actions**。

预期公开地址为：`https://jorsamj.github.io/shijuegushi/`。

入口音乐在用户首次点击“进入人生档案”后播放；它会在进入《雨夜来电》前与剧情 BGM 交叉淡出。静音和音量偏好存储在浏览器 localStorage 中。

## 数据校验

```bash
node scripts/validate-story-data.mjs
```
